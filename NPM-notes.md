### express

> _Є одним з найпопулярніших фреймворків для створення веб-додатків на платформі Node.js._\
> _Він надає набір інструментів та функціоналу, які спрощують розробку веб-додатків, API і веб-серверів._

**Ось ключові особливості та концепції, пов'язані з Express:**
+ Маршрутизація:
  - Express надає потужний механізм для визначення маршрутів (routes), які визначають, як ваш додаток відповідає на різні HTTP-запити (GET, POST, PUT, DELETE і т. д.).
  - Маршрутизація дозволяє вам визначити, які функції обробляють кожен тип запиту та URL-шляхи.
+ Middleware:
  - Express побудований навколо концепції middleware.
  - Middleware - це функції, які виконуються перед обробкою запиту.
  - Ви можете використовувати middleware для роботи з тілом запиту, аутентифікації, авторизації, обробки помилок і багато іншого.
  - Ви можете встановлювати готові middleware або писати власні.
+ Шаблонізатори:
  - Express дозволяє вам легко використовувати шаблонізатори, такі як Handlebars, EJS або Pug, для створення динамічних HTML-сторінок.
  - Ви можете генерувати HTML на основі даних, які відправляються на клієнтський браузер.
+ Статичні файли:
  - Ви можете налаштувати Express для обслуговування статичних файлів, таких як CSS, JavaScript, зображення і інші.
  - Це спрощує розгортання фронтенду в вашому додатку.
+ Робота з різними HTTP-методами:
  - Express підтримує всі основні HTTP-методи, такі як GET, POST, PUT, DELETE, і багато інших.
  - Ви можете визначити обробники для кожного з них.
+ Робота з параметрами і запитами:
  - Ви можете отримувати доступ до параметрів URL, запитів, тіла запиту і інших частин запиту в Express для обробки даних і прийняття рішень.
+ Сесії і аутентифікація:
  - Express надає можливості для роботи з сесіями і автентифікацією, що допомагає зберігати стан між запитами і захищати ваш додаток.
+ Пакетний менеджер:
  - Express підтримує легке встановлення і використання пакетів через npm, що дозволяє розширювати можливості вашого додатку.

* npm-site: https://www.npmjs.com/package/express
* install: `npm install express`
* home-page: https://expressjs.com/

> _Основними перевагами Express є його простота в використанні, широкий спектр розширень та активна спільнота, яка підтримує фреймворк._

### mongoose

> _це бібліотека для роботи з MongoDB у Node.js додатках._\
> _MongoDB - це нереляційна база даних, і Mongoose дозволяє вам зручно взаємодіяти з нею, використовуючи схеми (schemas) і моделі (models)._

**Ось ключові концепції та функціонал, пов'язані з Mongoose:**
+ Схеми (Schemas):
  - В Mongoose ви визначаєте структуру документів бази даних за допомогою схем.
  - Схема описує, які поля може мати документ, їх типи, валідацію та інші обмеження.
+ Моделі (Models):
  - Модель в Mongoose - це клас, який використовується для взаємодії з конкретною колекцією в MongoDB.
  - Модель відповідає за створення, зчитування, оновлення і видалення документів.
+ Підключення до бази даних:
  - Mongoose дозволяє вам встановити з'єднання з MongoDB-сервером і вибрати базу даних, з якою ви будете працювати.
+ Запити та операції з даними:
  - Ви можете використовувати моделі Mongoose для виконання різних операцій з даними, таких як пошук, оновлення, вставка і видалення.
+ Валідація даних:
  - Ви можете встановлювати правила валідації для полів у схемі, і Mongoose буде перевіряти дані перед збереженням їх у базу даних.
+ Підтримка Middleware:
  - Mongoose надає можливість встановлювати middleware для реагування на події, такі як перед збереженням документа або перед його видаленням.
  - Це дозволяє виконувати певні дії перед або після операцій з базою даних.
+ Параметризація запитів:
  - Ви можете використовувати різні методи Mongoose для створення складних запитів до бази даних, використовуючи вирази, умови і сортування.
+ Пагінація та сортування:
  - Mongoose дозволяє легко реалізувати пагінацію та сортування даних у ваших додатках.
+ Підтримка Promise та async/await:
  - Мongoose підтримує використання Promise і async/await для асинхронної роботи з базою даних.

* npm-site: https://www.npmjs.com/package/mongoose
* install: `npm install mongoose`
* home-page: https://mongoosejs.com/

> _Основними перевагами Mongoose є зручність роботи з MongoDB в контексті Node.js додатків, можливість валідації даних і роботи з ними, а також вбудовані можливості для обробки подій і middleware._

### dotenv

> _це npm-пакет, який допомагає завантажити конфігураційні змінні з файлу .env у змінні середовища вашого Node.js додатка._\
> _Цей пакет дозволяє вам уникнути розголошення конфіденційної інформації (наприклад, ключі API або паролі) відкрито в коді вашого додатка і зберігати їх в окремому файлі, який може бути доданий в .gitignore, щоб не потрапити в систему контролю версій._

* npm-site: https://www.npmjs.com/package/dotenv
* install: `npm install dotenv --save`
* home-page: https://github.com/motdotla/dotenv#readme

1. Створіть файл `.env` у кореневій папці вашого проекту.
     - _У цьому файлі ви можете визначити змінні середовища та їх значення._
     - ```plaintext
           DB_HOST=localhost
           DB_PORT=27017
           SECRET_KEY=mysecretkey
       ```
2. Ви можете використовувати `dotenv` для завантаження змінних середовища з файлу `.env` у вашому коді.
     - Для цього додайте код зверху вашого файлу Node.js:
     - ```javascript
         require('dotenv').config();
       ```
     - ви зможете отримувати доступ до змінних середовища (файл .env), так:
     - ```javascript
         const dbHost = process.env.DB_HOST;
         const dbPort = process.env.DB_PORT;
         const secretKey = process.env.SECRET_KEY;
       ```
3. Впевніться, що ви не включаєте файл `.env` в систему контролю версій, додавши його до файлу `.gitignore`. 

Цей підхід рекомендується для безпечного та зручного керування конфігурацією вашого Node.js додатка, особливо при роботі з секретними ключами або паролями.

### jsonwebtoken

> _це npm-пакет для генерації та перевірки JSON Web Tokens (JWT) в Node.js._
> _JWT - це стандартизований спосіб представлення структурованих даних у форматі, який можна передавати між сторонами як JSON об'єкт. JWT використовується для автентифікації і забезпечення безпеки в веб-додатках та API._

**Ось ключові концепції та функціонал, пов'язані з jsonwebtoken:**
+ Створення JWT:
  - Ви можете використовувати jsonwebtoken для створення JWT, які містять корисну інформацію (пейлоад), таку як ім'я користувача, роль, термін дії токена та інше.
  - Токен підписується секретним ключем, що дозволяє перевірити його цілісність.
+ Підпис та верифікація:
  - JWT можуть бути підписані за допомогою секретного ключа (симетричне шифрування) або за допомогою пари ключів (асиметричне шифрування).
  - Підпис дозволяє перевірити, чи були дані токена змінені після його створення.
+ Термін дії токена:
  - JWT може мати обмежений термін дії, після якого він стає недійсним.
  - Це дозволяє контролювати тривалість сеансу автентифікації.
+ Пейлоад (Payload):
  - Це дані, які зазвичай містяться в JWT.
  - Пейлоад може бути користувачем, ролями, датами та іншою інформацією.
  - Важливо помітити, що пейлоад не є конфіденційними, і вони можуть бути прочитані клієнтом.
+ Захист від підробки:
  - Завдяки підпису і, в деяких випадках, шифруванню, JWT захищаються від підробки.
  - Якщо хтось спробує змінити дані токена, підпис стане недійсним.
+ Використання на стороні клієнта та сервера:
  - JWT можуть бути використані для автентифікації користувачів на стороні сервера та для авторизації запитів на стороні клієнта.
  - Клієнт може зберігати JWT у себе і передавати їх на сервер для доступу до захищених ресурсів.

* npm-site: https://www.npmjs.com/package/jsonwebtoken
* install: `npm install jsonwebtoken`
* home-page: https://github.com/auth0/node-jsonwebtoken#readme

**Ось приклад створення та перевірки JWT за допомогою jsonwebtoken:**
```javascript
  const jwt = require('jsonwebtoken');

  // секретний ключ для підпису та верифікації JWT
  const secretKey = 'secret';

  // дані для пейлоаду.
  const payload = {
    userId: 123,
    username: 'exampleuser'
  };

  // створення JWT з пейлоадом та підписом
  const token = jwt.sign(payload, secretKey, { expiresIn: '1h' });

  console.log('Generated JWT:', token);

  //перевірка та розкодування JWT
  jwt.verify(token, secretKey, (err, decoded) => {
    if (err) {
        console.error('JWT verification failed:', err.message);
    } else {
        console.log('Decoded JWT:', decoded);
    }
  });
```

> _Важливо зберігати секретний ключ в безпечному місці, оскільки він використовується для підпису та верифікації токенів._

### body-parser

> _це npm-пакет для обробки даних, які надходять у тілі HTTP-запитів у вашому Node.js додатку, особливо при використанні фреймворка Express.js._\
> _Цей пакет розширює можливості обробки даних, дозволяючи вам легко отримувати доступ до даних, що надходять в запитах POST, PUT, PATCH і т.д._

**Основні функції та концепції body-parser:**
+ Обробка даних з тіла запиту:
  - body-parser допомагає вам отримувати доступ до даних, які надходять у тілі HTTP-запитів.
  - Це особливо важливо для обробки даних форми, JSON-даних та інших типів даних, що надходять в POST або PUT запитах.
+ Підтримка різних типів даних:
  - Пакет підтримує різні типи даних, включаючи JSON, URL-кодовані дані (форми) та інші.
  - Ви можете легко налаштовувати, які типи даних обробляти.
+ Легка інтеграція з Express.js:
  - body-parser розроблений для інтеграції з фреймворком Express.js, і ви можете легко використовувати його в вашому Express додатку.
  - Він додає спеціальні об'єкти req.body до об'єкта запиту, де зберігаються оброблені дані з тіла запиту.
+ Параметри конфігурації:
  - body-parser має параметри конфігурації, які дозволяють вам налаштовувати його роботу відповідно до ваших потреб.
  - Наприклад, ви можете визначити максимальний розмір даних, який може бути оброблений.
+ Підтримка обробки файлів:
  - Пакет також надає можливість обробки файлів, які надходять через запити з використанням multipart/form-data.

* npm-site: https://www.npmjs.com/package/body-parser
* install: `npm install body-parser`
* home-page: https://github.com/expressjs/body-parser#readme

**Ось приклад використання body-parser у веб-додатку, створеному з використанням Express.js:**
```javascript
  const express = require('express');
  const bodyParser = require('body-parser');

  const app = express();

  // встановлюємо body-parser як middleware для обробки JSON-даних
  app.use(bodyParser.json());

  // обробка POST-запиту, де дані у форматі JSON
  app.post('/api/data', (req, res) => {
    // дані доступні через req.body, і їх можна обробляти у вашому обробнику маршруту
    const data = req.body;
    console.log('Received data:', data);
    // ваш код обробки отриманих даних тут
    res.send('Data received.');
  });

  app.listen(3000, () => {
    console.log('Server is running on port 3000');
  });

```

body-parser є корисним інструментом для обробки даних у вашому Node.js додатку та особливо важливим для обробки форм і данних, які надходять через HTTP-запити.

### bcryptjs

> _це npm-пакет для шифрування паролів у Node.js._\
> _Надає безпечний спосіб зберігання та порівняння хешів паролів, що робить його корисним для роботи з паролями в програмах, особливо в автентифікаційних системах._

**Основні функції та концепції bcryptjs:**
+ Хешування паролів:
  - Головна функція - це створення хешів паролів.
  - Замість зберігання паролів у чистому тексті у базі даних, ви може застосовувати бібліотеку для створення хешів паролів перед збереженням їх у базі даних.
+ Солі (Salt):
  - bcryptjs автоматично створює сіль для кожного паролю, що підвищує безпеку.
  - Сіль - це випадковий рядок, який додається до паролю перед хешуванням.
  - Це робить атаки з використанням таблиць реґістрації (rainbow tables) надзвичайно важкими.
+ Порівняння паролів:
  - bcryptjs надає метод для порівняння паролів у вигляді звичайного тексту з хешами у базі даних.
  - Ви можете використовувати цей метод для перевірки правильності введеного користувачем пароля під час процесу автентифікації.
+ Параметри хешування:
  - Ви можете налаштовувати параметри хешування, такі як вартість (cost), яка визначає кількість ітерацій для генерації хеша. Збільшення вартості зробить атаку методом перебору більш витратною для зловмисників.
+ Простота використання:
  - Ця бібліотека досить легка в використанні. Вам потрібно лише імпортувати бібліотеку, і ви готові створювати та порівнювати хеші паролів.

* npm-site: https://www.npmjs.com/package/bcryptjs
* install: `npm install bcryptjs`
* home-page: https://github.com/dcodeIO/bcrypt.js#readme

**Щоб створити хеш паролю, ви можете використовувати наступний код:**
```javascript
  const bcrypt = require('bcryptjs');
  // встановлюємо вартість (кількість ітерацій) для хешування паролю
  // це значення може варіюватися в залежності від потреб безпеки
  const saltRounds = 10; 
  // задаємо пароль у відкритому (незашифрованому) вигляді
  const plaintextPassword = 'password123';

  // генеруємо сіль (salt) для застосування до паролю
  bcrypt.genSalt(saltRounds, function(err, salt) {
    // тут ми отримали сіль, і тепер можемо використовувати її для хешування паролю
    // 1-м аргументом є пароль у відкритому вигляді, 2-м - сіль
    bcrypt.hash(plaintextPassword, salt, function(err, hash) {
      if (err) {
        console.error(err);
      } else {
          // тут ми отримали хеш паролю
            // і можемо зберегти цей хеш у базі даних
              console.log('Hashed Password:', hash);
          }
      });
  });
```

> _Зверніть увагу, що бібліотека пропонує асинхронний підхід до хешування для запобігання блокуванню Node.js в процесі обчислення хешів._

bcryptjs - це потужний інструмент для забезпечення безпеки паролів у вашому Node.js додатку, і його варто використовувати для будь-якого проекту, де важлива безпека автентифікації користувачів.

### nodemon

> _це корисний інструмент для розробників Node.js, який спрощує процес розробки, особливо під час роботи з серверними додатками._\
> _Основна особливість Nodemon - це автоматична перезавантаження серверного додатку при зміні вихідного коду._

**Ось докладніше про цей npm-пакет:**
+ Автоматичне перезавантаження:
  - Nodemon стежить за змінами файлів в вашому проекті.
  - Якщо він виявляє зміни в коді (наприклад, ви зберегли файл), він автоматично перезапускає ваш сервер, щоб відобразити зміни без необхідності вручному перезапуску сервера.
+ Підтримка для різних мов та фреймворків:
  - Nodemon підтримує не лише Node.js, але і інші мови програмування, такі як Python, Ruby, і т.д.
  - Ви можете використовувати його з будь-якими фреймворками чи бібліотеками, що підтримують автоматичне перезавантаження.
+ Корисні параметри командного рядка:
  - Nodemon дає вам можливість настроїти його роботу за допомогою параметрів командного рядка.
  - Ви можете вказати, які файли слід відслідковувати, які файли ігнорувати, і багато іншого.
+ Розширена конфігурація:
  - Поза параметрами командного рядка, Nodemon також підтримує конфігурацію через файл nodemon.json або через поле nodemonConfig у вашому package.json.
  - Це дозволяє вам настроїти його більш докладно.
+ Плагіни та розширення:

  - Це дає можливість додавати специфічний функціонал до Nodemon для вашого проекту.

* npm-site: https://www.npmjs.com/package/nodemon
* install: `npm install --save-dev nodemon`

> _Після цього ви можете запустити ваш сервер за допомогою nodemon, замість node, і Nodemon відслідковуватиме зміни та автоматично перезапускатиме його при необхідності._

### morgan

> _це middleware для Node.js, який використовується для створення журналу (логу) запитів HTTP;_\
> _Він є дуже популярним і корисним інструментом для відстеження та аналізу HTTP-запитів, що надходять до сервера Node.js._

**Основні риси 'morgan' включають:**
+ Легка інтеграція:
  - 'morgan' легко інтегрується в твій серверний код за допомогою npm.
+ Різні формати журналу:
  - надає можливість настроювати формат виводу журналу, щоб відобразити саме ту інформацію, яка тобі потрібна.
+ Підтримка різних рівнів журналювання:
  - Ви можете вибирати між різними рівнями журналювання, такими як 'combined' (стандартний формат журналу) і 'dev' (зручний для розробників формат).
+ Автоматична обробка запитів:
  - 'morgan' автоматично додає інформацію про кожний HTTP-запит, таку як метод, URL, статус і час виконання, до журналу.
+ Підтримка кастомного форматування:
  - Ви можете налаштувати 'morgan' для виведення будь-якої додаткової інформації або логіку форматування.

* npm-site: https://www.npmjs.com/package/morgan
* install: `npm install morgan`
* home-page: https://github.com/expressjs/morgan#readme

Після встановлення та налаштування 'morgan' в твоєму Node.js додатку, він буде автоматично журналювати запити, що надходять до сервера, що допоможе тобі відстежувати роботу твого додатку та виявляти можливі проблеми.

#### Example: vanilla http server
> _Simple app that will log all request in the Apache combined format to STDOUT_

```node
  var finalhandler = require('finalhandler')
  var http = require('http')
  var morgan = require('morgan')
 
  // create "middleware"
  var logger = morgan('combined')
 
  http.createServer(function (req, res) {
    var done = finalhandler(req, res)
    logger(req, res, function (err) {
      if (err) return done(err)
 
      // respond to request
      res.setHeader('content-type', 'text/plain')
      res.end('hello, world!')
    })
  })
```

### react-router-dom
> _це бібліотека для навігації між сторінками та управління маршрутами в веб-додатках, що використовують React._\
> _Вона дозволяє створювати односторінкові додатки (Single Page Applications, SPA) та робити маршрутизацію між різними частинами вашого додатку без перезавантаження сторінки._

**Основні концепції та компоненти:**
+ BrowserRouter:
  - Це компонент, який навколює ваш додаток і встановлює контекст для роботи з маршрутами.
+ Route:
  - Використовується для визначення, яка компонента повинна рендеритися на основі поточного URL.
+ Link:
  - Компонент для створення посилань, які переходять між сторінками без перезавантаження сторінки.
+ Switch:
  - Використовується для групування маршрутів і вибору першого збігу з URL.
+ Redirect:
  - Використовується для перенаправлення користувача на іншу сторінку.
+ useHistory:
  - Кастомний хук, який дозволяє змінювати історію перегляду та переходити на інші сторінки програмно.
+ useParams:
  - Хук для отримання параметрів з URL.

> _react-router-dom дозволяє вам легко визначати маршрути в вашому додатку, надає функціонал для взаємодії з URL та визначення, яка компонента має бути показана на поточній сторінці. Це надає користувачам інтерактивний досвід перегляду вашого додатку, дозволяючи їм переміщатися між різними сторінками без перезавантаження сторінки._

* npm-site: https://www.npmjs.com/package/react-router-dom
* install: `npm i react-router-dom`
* home-page: https://reactrouter.com/en/main

Важливо враховувати, що react-router-dom є однією з найпопулярніших бібліотек для роботи з маршрутизацією в React-додатках і її використання робить процес навігації та створення SPA більш зручним та ефективним.

### web-vitals
> _це бібліотека, яка надає інструменти для вимірювання та моніторингу ключових вітальних показників (web vital metrics) веб-сторінок. Веб-вітальні показники - це метрики, які вимірюють важливі аспекти швидкості та реакції веб-сторінки під час її завантаження та взаємодії користувача._

**Основні веб-вітальні показники включають такі метрики:**
+ Largest Contentful Paint (LCP):
  - Вимірює час, який потрібен, щоб завантажити та відобразити найбільший вміст на сторінці.
  - Це важливо для сприйняття користувачем швидкості завантаження сторінки.
+ First Input Delay (FID):
  - Вимірює час, який потрібен для реакції сторінки на перший вхід користувача, наприклад, клік чи натискання на клавішу.
  - Це важливо для оцінки реакції сторінки на взаємодію користувача.
+ Cumulative Layout Shift (CLS):
  - Вимірює, наскільки сторінка "пересувається" під час завантаження через відмінності у розміщенні елементів.
  - Високий CLS може призвести до незручностей для користувача, якщо елементи переміщаються, коли він намагається взаємодіяти зі сторінкою.

> _web-vitals надає API для вимірювання цих метрик на вашому сайті та створює звіти, які допомагають вам зрозуміти, наскільки ваша сторінка відповідає стандартам щодо швидкості та реакції._

* npm-site: https://www.npmjs.com/package/web-vitals
* install: `npm i web-vitals`
* home-page: https://web.dev/vitals/
